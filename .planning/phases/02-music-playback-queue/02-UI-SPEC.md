## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, minimal padding |
| sm | 8px | Compact spacing, label margins |
| md | 16px | Default element spacing, container padding |
| lg | 24px | Control button spacing, section padding |
| xl | 32px | Layout gaps, major spacing |
| 2xl | 48px | Player control button size (touch target) |

**Exceptions:** 
- Player control buttons: 48px touch target (accessibility requirement)
- Mini-player height: 64px (contains album art + controls)

## Interaction Patterns

### Player Layout
```
┌─────────────────────────────┐
│     Album Art (300x300)     │
│                             │
├─────────────────────────────┤
│  Track Title (20px/600)     │
│  Artist Name (14px/400)     │
├─────────────────────────────┤
│  0:34 ━━━●━━━━━━━━━ 3:42    │  ← Progress bar (draggable)
├─────────────────────────────┤
│  [shuffle] [prev] [PLAY] [next] [repeat]  │  ← Controls (48px touch)
│     16px      24px    24px    24px    16px   │  ← Spacing
└─────────────────────────────┘
```

**Control Order (left to right):**
1. Shuffle toggle (left edge)
2. Previous track
3. Play/Pause (center, primary)
4. Next track
5. Repeat toggle (right edge)

### Mini-Player (64px height)
```
┌──────────────────────────────────────┐
│ [Art] Track Title     [Prev][Play][Next] │  ← 64px container
│ 48px  Artist Name     48px  48px   48px │  ← Touch targets
└──────────────────────────────────────┘
```

### Queue Item (64px height)
```
┌──────────────────────────────────────┐
│ [Grip] [Art] Track Title      [Remove] │  ← Drag handle + remove
│  16px  48px  Artist • 3:42      32px │  ← Icons
└──────────────────────────────────────┘
```

**Queue Reordering:**
- Long-press on queue item activates drag mode
- Drag handle (≡) visible on left side
- Item lifts with shadow during drag
- Haptic feedback on reorder
- Auto-scroll when dragging near edge

### Progress Bar Seeking
- Tap to jump to position
- Drag thumb to scrub
- Show timestamp tooltip during drag
- Haptic tick every 5 seconds during scrub

## State Management Requirements

**Playback State:**
- Current track (Track object)
- Playback position (milliseconds)
- Duration (milliseconds)
- Playing state (boolean)
- Shuffle enabled (boolean)
- Repeat mode ('off' | 'all' | 'one')
- Loading state (boolean)

**Queue State:**
- Queue array (Track[])
- Current index (number)
- Original order (for shuffle restoration)

**Context/Provider:**
- Create `PlaybackContext` to share state across components
- Mini-player visible on all screens (except player itself)
- Persist queue and playback position to AsyncStorage

## Accessibility Requirements

**Touch Targets:**
- All interactive controls: minimum 48px × 48px
- Queue item remove button: 48px × 48px
- Progress bar thumb: 48px touch area (visual can be smaller)

**Screen Reader:**
- All controls have descriptive labels
- Progress bar announces current time and remaining time
- Queue items announce track name, artist, and position in queue

**Color Contrast:**
- Light mode: All text meets WCAG AA (4.5:1 for body, 3:1 for large)
- Dark mode: Text on #121212 background meets AA standards
- Accent #007AFF has sufficient contrast on both backgrounds

**Visual Feedback:**
- Button press: opacity 0.7 on TouchableOpacity
- Active states: Accent color background
- Loading: ActivityIndicator with appropriate contrast
- Error: Destructive color #ff3b30 with clear messaging

## Animation Specifications

**Player Transitions:**
- Mini-player to full player: 300ms ease-in-out slide up
- Full player to mini-player: 300ms ease-in-out slide down
- Album art rotation (if playing): 20s linear infinite

**Queue Interactions:**
- Item reorder: Smooth spring animation (tension: 100, friction: 20)
- Item remove: 200ms fade out + slide left
- Item add: 200ms fade in + slide down

**Progress Bar:**
- Thumb scale on drag: 150ms ease-out (1.0 → 1.2 scale)
- Bar fill: Real-time update (no animation, direct position)

**Button States:**
- Press: 100ms opacity change (1.0 → 0.7)
- Toggle state change: 200ms cross-fade between icons

---

## Performance Constraints

**Queue Rendering:**
- Use FlatList with `windowSize={10}` for efficient rendering
- `getItemLayout` for consistent 64px item height
- `keyExtractor` using track ID

**Progress Bar:**
- Update max 10 times per second (avoid 60fps updates)
- Debounce seek events (50ms) to prevent excessive API calls

**Image Loading:**
- Album art: Cache with expo-image or react-native-fast-image
- Placeholder: Show gray rectangle during load
- Max resolution: 300x300 for player, 48x48 for mini-player and queue

**Memory:**
- Queue limit: 1000 tracks (per REQUIREMENTS.md)
- Unload full player album art when minimized

## Platform-Specific Considerations

**iOS:**
- Use native haptic feedback (Haptics.impactAsync)
- Lock screen controls via expo-av MPNowPlayingInfoCenter

- Support CarPlay if possible (future enhancement)

**Android:**
- Use Vibration API for haptic feedback

- Respect Do Not Disturb mode

- Background audio capability required
- Handle interruptions (phone calls, alarms)
- Audio focus management (duck/pause on interruption)

## Notes

**Design System Context:**
This is a React Native mobile application, not a web app. shadcn/ui and web-based component libraries are not applicable. All UI is built using React Native's built-in components (View, Text, TouchableOpacity, FlatList, etc.) and styled with StyleSheet.create().

**Consistency with Phase 1:**
- Reuses existing color palette (#007AFF primary, #ff3b30 destructive)
- Extends typography with two specifications: 20px/600 for track titles, 400 regular weight for body text (Phase 1 only used 600 semibold)
- Maintains 8px spacing grid established in Phase 1
- Continues pattern of icon-only buttons (established in auth screens)

**Dark Mode Implementation:**
Light/dark mode will be detected using `useColorScheme()` hook from React Native. Color values will be selected based on the current scheme. All components must support both modes from the start.

**Icon Specifications:**
Use Ionicons from @expo/vector-icons:
- Play: "play-circle" (filled), 48px
- Pause: "pause-circle" (filled), 48px
- Next: "play-skip-forward", 32px
- Previous: "play-skip-back", 32px
- Shuffle: "shuffle", 24px
- Repeat: "repeat", 24px
- Remove: "close-circle", 24px
- Drag handle: "reorder-three", 20px

All icons use accent color (#007AFF) when active, text secondary color when inactive.
